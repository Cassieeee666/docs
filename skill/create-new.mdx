## Step 1: Start a New Skill

Click **Add New Skill** to begin creating a skill from scratch.

## Step 2: Define Basic Information

Set the **skill name , icon** and select the **Application channel**.

- **Email Skills:** Suitable for handling customer complaints received via the Email channel.
- **Livechat Skills:** Suitable for handling real-time conversations from the Livechat channel.
- **Inbound Voice:** Suitable for handling incoming phone calls.

<Note>
  Skills of different channels include different configuration options. For details, please refer to **Step 4**.
</Note>

</Frame>

## Step 3: Configure Common Settings

Complete the shared configuration for all skill types:

- **When to use**

  Define the conditions under which the skill should be triggered.

  For example: _"When the customer asks about the order logistics status, like when the order will be shipped, where the package is currently, etc."_

  <Frame>
    ![Agent Whentouse](/agent-whentouse.png)
  </Frame>
- **Prompt**

  Write a structured skill prompt to define behavior and task logic.

  Use "`/`" to insert knowledge references, handoff actions, or tools.

  <Tip>
    Want to learn how to write a perfect prompt? See [Write a Skill Prompt](/skill/prompt)
  </Tip>
  <Frame>
    ![Writeskillprompt](/writeskillprompt.png)
  </Frame>
- **Tools**

  Add required tools and click **"Connect"** to complete authorization. You can add the following 5 tools:
  - **Logistics_inquiry:** Get logistics information with tracking numbers.
  - **google_calendar:** Check availability, as well as creating, editing, and deleting events.
  - **google_sheet:** This tool allows reading and writing data to spreadsheets.
  - **send_email:** Send Emails
  - **shopify:** Retrieve detailed order information using the order number, including tracking numbers, product details, and more.

  <Note>
    All tools except the logistics_inquiry tool need authorization—just follow the guide to jump and complete it in one click.

    See [Deploy Overview](/deploy/overview) for tool details.
  </Note>
  <Frame>
    ![Image](/images/image-17.png)
  </Frame>
- **Knowledge Access**

  Enable knowledge-based responses and add "**retrieve_knowledge**" tool in the Prompt.

  <Frame>
    ![Skill Add Knowledge](/skill-add-knowledge.png)
  </Frame>

## Step 5: Save and Publish

Click **Save** to store the skill.

<Warning>
  To make the skill active in production, please “publish” the agent. Changes will only take effect after the agent is published.
</Warning>
